Lecture: Loops

Imagine if we wanted to write to the console all the even numbers between 0 and 100. Or even just repeat the same thing -- like "text my mom something nice" or "text my ex something mean" -- many times?

We could simply copy/paste many console.log commands: 

console.log(1)
console.log(2)
console.log(3)
console.log(4)
console.log(5)
console.log(6)
console.log(7)
console.log(8)
console.log(9)
console.log(10)
console.log(11)
console.log(12)
console.log(13)
console.log(14)
console.log(15)
// etc ...

But that looks pretty tedious! What if we wanted to do this 100, or 1,000, or 1,000,000 times?! That would be a lot of copy/pasting.

What if we wanted to prompt the user to input a number, and print something that many times? There is no way we could do this simply by copy/pasting!

Computers are not very smart, but they are very good at doing repetetive tasks. How? They use loops! Many algorithms depend on this type of behavior.

Loops use several things that we have already talked about (variables, arithmetic and conditionals) and put them together with a new while keyword to instruct the computer how to do repetive tasks. 

var i = 0
while ( i < 100 ) {
  console.log(i)
  i = i + 1
}
Read this as "Set i to 0. While i is less than 100, print i to the console and increase i by one."

So what's going on here? Let's break it down:

The loop has four major sections:

  1. Initialization: var i = 0
  2. Condition: i < 100
  3. Body: 
    console.log(i)
i = i + 1
  4. This last line has a special purpose so we give it a special name: the Increment.

In each of these four sections, you can put in any values that you need to achieve the kind of repetition that you are trying to implement. (Or you can put in nothing at all! but the vast majority of the time all four sections have something in them.)

Sometimes you might create a loop that never completes. This happens when the condition is never met. We call this an "infinite loop." (Now do you get the joke of the address of Apple's Headquarters in Cupertino, CA?) There are some occasions where you might want this, but for now it is a bad thing. For example, if you forget to include an Increment, your looping variable will never increase, your stopping condition will never be met, and your page will hang. Let's try this once to see what happens and how to get out of it. [WARNING: save your work first!] Now in general: be careful not to do this!

Initialization

The initialization is where you set up the variables that you need for your loop. The most common is var i = 0; but you can start at anything, for example: 1 or 1000000, depending on what your program is doing.

For example, the following loop will print each year since the invention of the World Wide Web. 

var i = 1989;
while ( i < 2017 ) {
  console.log(i);
  i = i + 1;
}

Challenge Question: Try changing the variable initialization so that the code prints each year since you were born.

Condition

A loop will execute the statements in the body repeatedly until the condition evaluates to false. Each execution of the body is called an iteration.

Challenge Question: Try changing the above example so that no matter what the initial variable value is, the code prints the next 10 years.
Hint 1: You can use basic variable arithmetic in the condition statement.
Hint 2: Be careful to avoid an infinite loop here! You might need to introduce a 2nd variable to make this work.

Increment

The increment is where you update the variables at the end of each iteration. The most common is i = i + 1 (or i++) but you can do anything, for example: i += 2 or i *= 2, depending on what your program is doing.

The following loop prints all even numbers between 10 and 20. 

var i = 10;
while ( i < 20 ) {
  console.log(i);
  i = i + 2;
}

Challenge Question: Try changing the above example so that it prints all the years since you've been born but in reverse order, starting with this year first and going back in time.
Hint: the "increment" can actually decrease the variable value.

Body

The body is the meat of your loop. The rest is just figuring out where you start, where you end, and what the step size is. The body gets executed over and over again until the condition evaluates to false.

The body of a loop can include any valid JavaScript statements.

Challenge Question: Try modifying the example that prints each year since you've been born, but also prints "Can't vote" or "Can vote!" depending on whether you were at least 18 that year.
Hint: Conditionals are perfectly valid in the body of a loop.

Syntactic Sugar

Similar to the way that English allows multiple ways to say the same thing, most programming languages have syntax rules that allow you to write the same instructions in different ways. One example of this is the way you can write the Initialization, Condition and Increment of a while loop in one compact line with a bit of equivalent syntax called a for loop. The following two code snippets are exactly equivalent: 

var i = 0;
while ( i < 100 ) {
    console.log(i);
    i = i + 1;
  }
and 
for ( var i = 0; i < 100; i = i + 1 ) {
    console.log(i);
  }
Most experienced programmers tend to write loops using the for structure, but if you find while loops to be more easily readable, feel free to use which ever you like!
